ProofAtlas

Recall-oriented candidate generation for Lean premise selection

ProofAtlas substantially improves recall-oriented premise candidate generation by fusing proof-state retrieval with LLM-enriched theorem-neighborhood evidence.

Covered Recall is computed over retrievable gold premises. All-positive Recall is computed over all held-out positive edges.

Dataset & Evaluation Scope

Task: given a held-out proof state, retrieve useful premises from 127,561 train-side candidate premises.

Split Theorems Proof states Premises Positive edges
train 8000 23723 127561 54897
test 1000 3053 38332 7054
Retrieval pool 127,561 train-side premises
Query set 3,053 held-out proof states
Metric Recall@100 = gold premise coverage within top 100 candidates

Train/test are theorem-disjoint. Covered Recall is over retrievable gold premises; All-positive Recall is over all held-out positive edges. This is not a theorem-statement-only setting.

System Pipeline

LLM enrichment converts theorem metadata plus proof-state goals, hypotheses, and symbols into semantic, strategy-oriented, and difficulty-oriented theorem profiles. These profiles feed theorem-neighbor premise evidence, which is fused with proof-state retrieval by weighted reciprocal-rank fusion.

LLM enrichment is retrieval text only; it does not use gold premise labels or proof scripts as target answers.

The primary method is the strongest non-BGE LLM theorem-neighborhood variant; BGE fusion is reported as an auxiliary recall-maximizing variant.

Held-Out Results

ProofAtlas improves recall-oriented premise candidate generation. The primary method is the strongest non-BGE LLM theorem-neighborhood variant; BGE fusion is shown separately as an auxiliary recall-maximizing variant. Top-rank precision remains a future reranking problem.
